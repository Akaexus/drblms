Podstawowe

Node.js

drblms został napisany w JavaScript z użyciem Node.js. Minimalna wersja, na której zadziała to v4.0.0 [Source Code] (z npm v2.14.2) ze względu na pełne wsparcie dla ECMAScript 6 (wcześniejsze wersje wymagały dodatkowych flag do działania). Rekomendowana wersja Node.js do działania drblms to v6.10.2 LTS [Source Code] (z npm v3.10.10). Projekt zakończył również pozytywnie testy na Node.js z gałęzi 7.x (z npm 3.10.8 & 4.x).

  • Github
  • nodejs.org
Typ Wersja Node.js Wersja npm
Minimalna 4.0.0 2.14.2
Rekomendowana 6.10.2 3.10.10
Najnowsza 7.9.0 4.2.0

blessed

Projekt korzysta z pakietu blessed do rysowania interfejsu aplikacji. Minimalna obsługiwana wersja to 0.1.16 [Source Code] (wersja 0.1.15 również działa, jednak ma problemy z wyjściem z menu). Rekomendowana wersja to 0.1.81 [Source Code].

  • Github
  • npmJS
Typ Wersja blessed
Minimalna 0.1.6
Rekomendowana 0.1.81

ini

DRBL korzysta z plików ini (drblpush.conf) do przechowywania konfiguracji. Pakiet drblms korzysta z modułu ini do manipulacji tymi plikami. Minimalna obsługiwana wersja to [v1.0.0] [Source Code].

  • npmJS

Minimalna 1.0.0

Tworzenie dokumentacji

JSDoc

Projekt jest dokumentowany przy użyciu JSDoc. Dokumentowanie odbywa się poprzez dodawanie specjalnie sformatowanych komentarzy w kodzie.

Typ Wersja jsdoc
Minimalna 3.4.3

Tworzenie dokumentacji można wykonwać przez uruchomienie skryptu npm:

npm run doc

lub bezpośrednie uruchomienie jsdoc:

./node_modules/.bin/jsdoc -c jsdoc.json .

Instalacja Node.js

Arch Linux

pacman -S nodejs npm

Debian

NodeJS 6.x

curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -
sudo apt-get install -y nodejs

NodeJS 7.x

curl -sL https://deb.nodesource.com/setup_7.x | sudo -E bash -
sudo apt-get install -y nodejs

RHEL/Fedora

NodeJS 6.x

curl --silent --location https://rpm.nodesource.com/setup_6.x | bash -
yum -y install nodejs

NodeJS 7.x

curl --silent --location https://rpm.nodesource.com/setup_7.x | bash -
yum -y install nodejs

Dla Fedora 18

sudo dnf install nodejs

OS X

brew install node

Instalacja drblms

Plik *.tar.gz należy wypakować do dowolnej lokacji, a następnie wewnątrz wypakowanego katalogu wywołać npm:

tar -zxvf drblms.tar.gz
cd drblms
sudo npm install -g

Node Package Manager (npm) zadba o ściągnięcie wszystkich zależności w odpowiednich wersjach oraz instalacje drblms tak, jak robi systemowy menadżer pakietów. Po pomyślnej instalacji archiwum oraz wypakowany katalog można usunąć.

Dezinstalacja drblms

sudo npm uninstall -g drblms

Node Package Manager (npm) zadba o usunięcie wszystkich zależności, symlinków oraz plików należących do drblms.
